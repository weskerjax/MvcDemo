jQuery.tmpl()

No More String Concatenation

Templating â€“ why?

  • Avoid PHP-style mix of logic, content and presentation
  • Keep content out of the layout
  • Separation of concerns

Why client-side templating?

  • Keep UI responsive
  • Sending HTML updates via Ajax is inefficient
  • Faster computers and interpreters
  • Makes sense to do more work in the browser

Why jQuery.tmpl()?

  • jQuery has the biggest market share of any JS library at the moment
  • jQuery.tmpl() is its official templating plugin
  • Developed by Microsoft, under MIT/GPL licence

How does it work? (1)

Start with some data:

var person = { name: 'Dave' };

How does it work? (2)

Make a template for that data:

<script id='my-first-template'
  type='text/x-jquery-tmpl'>
  <p>Hello ${name}!</p>
</script>

How does it work? (3)

Combine the data with the template and add to the page:

$('#my-first-template')
  .tmpl(person).appendTo('body');

Template tags

  • ${} or {{=}}
  • {{html}}
  • {{if}} / {{else}}
  • {{each}}
  • {{tmpl}}
  • {{wrap}}

${}

Used for inserting data on the page.

//data:
var data = { name: 'Dave',
  cat: { name: 'Sparky' } };
<!--template:-->
<div>
  <p>My name is ${name} and my
  cat is called ${cat.name}</p>
</div>
<!--alternative syntax:-->
<p>Welcome back {{= name}}!</p>

${}

It is possible call simple functions and methods in ${}.

//custom function:
function fun(str) { return str + 'y'; }
<!--template:-->
<div>
  <p>I SAID MY NAME IS ${name.toUpperCase()}!</p>
  <p>But you can call me ${fun(name)}</p>
</div>

{{html}}

${} escapes HTML, but {{html}} passes it through untouched.

//data:
{ greeting:
  'Hello! How <em>nice</em> to see you!'
};
<!--template:-->
<p>{{html greeting}}</p>

{{if}} / {{else}}

Used for conditional execution of template code.

//data:
var data = { name: 'Dave', important: true };
<!--template:-->
{{if important}}
  <p>It's our most valued customer, ${name}!</p>
{{else}}
  <p>Hi there ${name}.</p>
{{/if}}

{{each}}

Used to iterate over an array within a template.

//data:
var data = { numbers: [4, 8, 15, 16, 23, 42] };
<!--template:-->
{{each numbers}}
  <p>${$index + 1}. Please enter the number ${$item} in your terminal.</p>
{{/each}}

{{tmpl}}

Used to render sub-templates. Can be passed a single item or an array.

var data = { name: 'Dave',
  cats: ['Mittens', 'Fluffy', 'Bob'] };

<script id='inner' type='text/x-jquery-tmpl'>
  <p>${$data}</p>
</script>
<script id='outer' type='text/x-jquery-tmpl'>
  <p>My name is ${name} and these are my cats:</p>
  {{tmpl(cats) '#inner'}}
</script>

{{wrap}}

//data
var data = { name: 'Dave' };
<!--wrapper template-->
<script id='wrapper' type='text/x-jquery-tmpl'>
  <div class='person-wrapper'>
    {{html $item.html()}}
  </div>
</script>
<!--wrapped template-->
<script id='wrapped' type='text/x-jquery-tmpl'>
  {{wrap '#wrapper'}}
    <p>${name}</p>
  {{/wrap}}
</script>

Template meta-variables

There are some special variables that can be used within templates:

  • $data: The current data item being processed
  • $item: The current template item
  • $value: The current value in {{each}}
  • $index: The current index in {{each}}

The three plugins

Each of these is available to be called on a jQuery instance or as a static method â€“ $(selector).tmpl() or $.tmpl()

  • jQuery.tmpl(): For rendering content in a specified template
  • jQuery.tmplItem(): For selecting and manipulating a rendered template item
  • jQuery.template(): For compiling a template for later or multiple use
